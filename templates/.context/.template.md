# Context Template for Projects

Este template define a estrutura `.context/` que deve existir em cada projeto.

## Estrutura

```
projeto/.context/
├── lessons-learned.md      # Lições acumuladas do projeto
├── decisions.md            # Decisões importantes tomadas
├── patterns.md             # Padrões descobertos neste projeto
├── feedback.md             # Feedback coletado
└── metrics/
    ├── phase-times.json    # Tempo gasto em cada fase
    └── issues-encountered.json  # Problemas encontrados
```

## Uso

### Criar Estrutura
```bash
mkdir -p .context/metrics
# Copiar templates acima
```

### Atualizar Durante Projeto
```yaml
A cada fase:
  1. Atualizar lessons-learned.md
  2. Registrar decisões importantes em decisions.md
  3. Anotar padrões em patterns.md
  4. Coletar feedback em feedback.md
  5. Atualizar métricas

Ao final de cada fase:
  1. Executar /retrospectiva
  2. Consolidar aprendizados
  3. Propor melhorias ao framework
```

### Benefícios

1. **Continuidade**: Se alguém pegar o projeto, entende o histórico
2. **Aprendizado**: Próximos projetos se beneficiam
3. **Melhoria do Framework**: Padrões viram features
4. **Auditoria**: Decisões ficam documentadas

---

**Nota**: A estrutura `.context/` é alimentada automaticamente pelas retrospectivas e pelo pattern detector do framework.
